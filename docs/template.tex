\section{Theoretical questions \points{1.5}}

\begin{enumerate}
  \item[(a)] \textbf{What are an eigenvector, an eigenvalue and an eigenbasis?}
  \\[.5em]
  \textbf{Eigenvectors} are non-null vectors that satisfy the following equation:
  $$
  \hat{A}\vec{x} = \lambda\vec{x}
  $$
  where $\hat{A}$ is a square matrix and $\lambda$ is a scalar number, meaning that the transformation of $\vec{x}$ performed by $\hat{A}$, produces a multiple of $\vec{x}$ by exactly $\lambda$ times. Knowing this definition, we can easily define that an \textbf{eigenvalue} is simply $\lambda$, that is, the real scaling coefficient for $\vec{x}$. Since $\lambda \in \mathbb{R}$, there exist an infinite number of eigenvectors for a bounded eigenvalue. To better group a family of eigenvectors, an \textbf{eigenbasis} is defined. An eigenbasis for $\lambda$ is defined as:
  $$
  E_{\lambda} = \{ \vec{v} ~ | ~ \hat{A}\vec{v} = \lambda \vec{v} \}
  $$
  
  \item[(b)] \textbf{What assumptions should be made to guarantee convergence of the power method?}
  \\[.5em]
  Having a matrix $\hat{A}$ of dimensions $n \times n$, $\hat{A}$ must have $n$ linearly independent eigenvectors so that the initial state vector (i.e. guess vector) can be expressed as a linear combination of those eigenvectors. This enables us to factor out the dominant eigenvalue and study the convergence since the other eigenvalues will eventually round off to zero. Therefore, there must be only a single dominant eigenvalue and the initial state vector must also have a non-zero component in the direction of the dominant eigenvalue.
  
  \item[(c)] \textbf{What is the shift and invert approach?}
  \\[.5em]
  The shift and invert approach is a technique that utilizes the inverse iteration algorithm to significantly increase the speed of convergence for eigenvalue computations. The shift consists in subtracting a shift factor $\alpha$ from the main diagonal of $\hat{A}$ so that the new eigenvalues become $(\lambda_j - \alpha)$, while the invert part inverts the matrix resulted from the latter operation $(\hat{A} - \alpha\hat{I})^{-1}$, thus giving the following eigenvalues:
  $$
  \mu_j = \frac{1}{\lambda_j - \alpha}
  $$
  By choosing a factor $\alpha$ that is very close to a specific target $\lambda_t$, the value $(\lambda_t - \alpha)$ becomes extremely small. This causes $\mu_j$ to become significantly more dominant than the other eigenvalues, reducing the number of iterations needed to perform the computation.
  
  \item[(d)] \textbf{What is the difference in cost of a single iteration of the power method, compared to the inverse iteration?}
  \\[.5em]
  The main difference in the cost comes from the power method using a matrix product as its primary operation, resulting in $O(n^2)$ time complexity, while the inverse iteration needs to solve linear systems implying $O(n^3)$ for factorization and $O(n^2)$ for substitution, resulting in $O(n^3)$ time complexity. Since the inverse iteration method converges faster, we will still perform computationally slower steps, but with a lower number of iterations. \label{qst:perf}
  
  \item[(e)] \textbf{What is a Rayleigh quotient and how can it be used for eigenvalue computations?}
  \\[.5em]
  The Rayleigh quotient is a scalar value that provides the best possible estimate for an eigenvalue given a corresponding vector. For a real matrix $\hat{A}$ and a vector $\vec{v}$, it is defined as:
  $$
  \mu(\vec{v}) = \frac{\vec{v}^T \hat{A}\vec{v}}{\vec{v}^T\vec{v}}
  $$
  In eigenvalue computations, it is used as an estimate tool, while in the Rayleigh Quotient Iteration, its value takes place as the shift $\alpha$ factor, making it dynamic.
  
\end{enumerate}

\section{Connectivity matrix and disjoint subgraphs \points{1}}
The connectivity matrix is a matrix representation of a graph that encodes the edges between nodes. An entry different from zero in the row $i$ and the column $j$ signifies a connection from node $i$ to node $j$. Since this representation encodes the direction of the edges, we can easily notice that a non-directed graph will produce a symmetrical matrix. This type of matrices inherently have many zero values, and since we are only interested in the edges, we can utilize sparse matrices to save memory space and represent only non-zero values.

\section{PageRanks by solving a sparse linear system \points{2.5}}
\subsection{Power method implementation}
In the \texttt{pagerank1.m} implementation of the power method, the stop condition is identified with a tolerance. By saving a previous state of $\vec{x}$, the error $\epsilon = |\vec{x}_k - \vec{x}_{k - 1}|$ can be computed and compared against an arbitrary tolerance $\tau$, in this case $0.1 \cdot 10^{-6}$, to stop the computation when the specified precision is reached.
\subsection{Inverse iteration method implementation}
The inverse iteration method in \texttt{pagerank2.m} continues to use a tolerance stop condition, but this time the Rayleigh Quotient $\mu(\vec{x})$ was used instead of the error $\epsilon$, defining the new error $\epsilon_R = |\mu(\vec{x})_k - \mu(\vec{x})_{k - 1}|$. Since the test is now on the eigenvalues, and by using the Rayleigh Quotient they converge faster than the eigenvectors, we need to prevent the algorithm to stop early by strengthening our tolerance. For this case, $0.1 \cdot 10^{-12}$ was used.
\subsection{Shift values}
After running the inverse iteration implementation with different $\alpha$ values, and checking the solutions, it is revealed that the vectors are not always the same. For $\alpha = 0.95$ and $\alpha = 1$, the algorithm converges to the identical and correct PageRank vector because both shift $\alpha$ values are closest to the dominant eigenvalue of 1. However, for $\alpha = 0.8$, the vector is completely different ($\sim0.1797$) because the shift is closer to the second eigenvalue ($0.85$), causing it to converge to the wrong eigenvector. For $\alpha = 0.9$, the method fails to converge at all, likely due to the oscillation between the closely clustered eigenvalues.
\subsection{Power vs. Inverse iteration}
Comparing the results obtained by calculating the PageRanks for the graphs \texttt{web1.m}, \texttt{web2.m}, and \texttt{web3.m} with the two implementations, it is noticed that both methods always converged with identical results except for the graph \texttt{web2.m}, where the power method ranked a different link with the same weight as third place. The power method always took a substantial larger number of iterations in comparison to the inverse iteration method. This is to be expected given what was discussed in \hyperref[qst:perf]{section 1, point d}. To check if the results were truly equal, the absolute distance of the two computation's output vectors was calculated, and the result found was always a numerical zero.
\\[2em]
The following advantages and disadvantages for the \textbf{power} method were identified:
\\[.5em]
\textbf{Advantages:}
\begin{itemize}
    \item Simplicity: it is straightforward to implement compared to the inverse iteration method
    \item Low Cost per Iteration: each step is $O(n^2)$, using basic matrix-vector multiplication. Given the sparsity of web graph matrices, this multiplication is fast and memory-efficient.
    \item High Scalability: it does not solve complex linear systems, making it the standard choice for massive graphs
\end{itemize}
\textbf{Disadvantages:}
\begin{itemize}
    \item More Iterations: requires significantly more iterations to converge compared to the inverse iteration method
    \item Convergence Rate Depends on $p$: the convergence speed is highly dependent on the damping factor $p$. As $p \to 1$, the power method takes increasingly longer to separate the dominant eigenvalue from the rest
\end{itemize}
\vspace{1.5em}
While for the \textbf{inverse iteration} method:
\\[.5em]
\textbf{Advantages:}
\begin{itemize}
    \item Less Iterations: by using a shift factor that is very close to the dominant eigenvalue, the algorithm hones in on the correct eigenvector almost immediately, resulting in extremely fast convergence
    \item Convergence Rate Depends Less on $p$: the algorithm is much less susceptible to slowdowns caused by high $p$ values
\end{itemize}
\textbf{Disadvantages:}
\begin{itemize}
    \item High Cost per Iteration: see \hyperref[qst:perf]{section 1, point d}
    \item Poor Scalability: solving massive, real-world like, linear system is computationally unfeasible. Matrix factorization destroys the sparsity of the matrix leading to massive memory consumption
    \item Sensitivity to $\alpha$ Choice: if $\alpha$ is chosen perfectly, the matrix becomes singular, this non-invertible
\end{itemize}

\section{Introduction to HPC: Algorithmic performance \points{1}}

\paragraph{Curve Growth and Theoretical Expectations}
Analyzing the execution time curves, both the Backslash operator and the Inverse Iteration method show a steep, non-linear growth as the matrix dimension $n$ increases. Specifically, the Inverse Iteration time escalates rapidly, reaching $469 \text{ ms}$ at $n=1500$. The Power Method instead, demonstrates a significantly flatter growth trajectory, peaking at only $35 \text{ ms}$. Regarding the number of iterations, the curves remain relatively stable across all matrix dimensions, yet they reveal a strong contrast in magnitude: the Power Method requires an average of $25$ to $30$ iterations, while Inverse Iteration converges in merely $7$ to $9$ steps. This behavior aligns perfectly with the expectations, as discussed in \hyperref[qst:perf]{section 1, point d}. For increasingly large matrices, the $\mathcal{O}(n^3)$ operations dominate the total execution time, explaining the rapid explosion in runtime despite the exceptionally low iteration count.

\paragraph{Scale Comparison and Visualization Advantages}
The primary advantage of the linear scale is its ability to convey absolute physical quantities. It effectively highlights the real-world computational penalty. The logarithmic scale, in this case semi-logarithmic, becomes highly advantageous when data points span multiple orders of magnitude. By representing relative changes rather than absolute increments, larger values do not take importance away from the smaller ones.

\paragraph{Utility of the Logarithmic Scale}
This scale is particularly crucial, in this case, for plotting the iterations. Because the Power Method requires roughly one order of magnitude more iterations than Inverse Iteration ($\sim 50$ vs. $\sim 8$), plotting these on a standard linear scale would compresses the Inverse Iteration data, making it appear as a flat line at zero. The logarithmic visualization proves that both methods maintain a stable iteration count independently of the matrix size $n$.
